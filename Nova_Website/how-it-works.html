Step by Step

Request lifecycle.

1
👤 User types or speaks a command
In the browser, the user types in the input box or clicks the 🎤 microphone button. The browser's SpeechRecognition API converts speech to text and fills the input box automatically.
// Browser: index.html + Web Speech API
const utterance = "search web Python tutorials";
2
📡 Browser sends a POST request to Flask
JavaScript's fetch() sends the message as JSON to the Flask backend at /process. This is an asynchronous call — the UI stays responsive while waiting.
fetch('/process', {
  method: 'POST',
  body: JSON.stringify({ message: "search web Python" })
})
3
⚡ Flask receives and routes the command
In app.py, the /process route extracts the message and passes it to process_command(). This function checks the command against keywords using if/elif logic to decide what to do.
@app.route("/process", methods=["POST"])
def process():
  data = request.get_json()
  return jsonify({ "response": process_command(data["message"]) })
4
🧠 Command handler runs the action
The appropriate function in commands.py is called. For example, search_web() opens Google in the browser, while search_wikipedia() calls the Wikipedia API and returns a summary.
def search_web(query):
  webbrowser.open(f"https://google.com/search?q={query}")
  return speak(f"Searching the web for: {query}")
5
🔊 Server speaks + returns text to browser
speak.py uses pyttsx3 to speak the response through the server's speakers (in terminal mode) AND returns the text string. Flask sends this text back as a JSON response.
def speak(text):
  engine.say(text)  # Speak on server
  engine.runAndWait()
  return text      # Return for browser
6
💬 Browser shows text + speaks response
JavaScript receives the JSON response, adds a message bubble to the chat, and calls SpeechSynthesis to speak the text aloud in the browser — giving you both visual and audio output.
const data = await response.json();
addMessage(data.response, 'nova'); // Show text
speakText(data.response);         // Speak it
Codebase

Project structure.

Nova follows a clean modular structure. Each file has one job, making it easy to understand and extend.

Nova_Assistant/
│
├── app.py ← Flask app & routing
├── requirements.txt ← pip dependencies
├── README.md ← usage guide
│
├── assistant/
│ ├── __init__.py ← package marker
│ ├── commands.py ← all command logic
│ ├── speak.py ← pyttsx3 TTS
│ └── utils.py ← time/date helpers
│
├── templates/
│ └── index.html ← chat web UI
│
└── static/
    └── style.css ← dark theme styles
Technology

What powers Nova.

🐍
Python 3
The core language. All logic, routing, and voice output is pure Python. No complex frameworks.
⚗️
Flask
Lightweight Python web framework. Serves the HTML page and handles API routes at /process.
🔊
pyttsx3
Offline text-to-speech. Works without internet. Cross-platform: Windows, Mac, Linux.
📖
Wikipedia API
Python library for fetching Wikipedia summaries. Clean, simple, and reliable.
🎤
Web Speech API
Built into Chrome/Edge browsers. No install needed. Converts voice input to text.
🌐
HTML / CSS / JS
Pure vanilla frontend. No React, no Vue. Beginner-friendly and fast.
